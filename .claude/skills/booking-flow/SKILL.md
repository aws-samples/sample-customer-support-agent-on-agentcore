# 约课操作流程

## 触发条件
用户表达约课意图，如：
- "我想约课"
- "帮我预约一节课"
- "明天有什么时间可以上课"
- "想给小明约一节中文课"

## 标准流程

### Step 1: 获取用户时区
调用 `get_parent_timezone` 获取用户所在时区，确保后续时间展示正确。

### Step 2: 查询可约时段
调用 `get_available_slots` 查询可用时间和老师：
- 如用户指定日期，查询该日期
- 如用户未指定，默认查询最近7天
- 如用户指定老师，使用 `tutor_id` 筛选

### Step 3: 向用户确认约课信息
展示可选时段并确认以下信息：
- **学生**：确认为哪个孩子约课
- **时间**：展示用户所在时区的时间
- **老师**：展示老师姓名和简介（如有）

**示例回复**：
```
好的，为您查到以下可约时段：

1. 2024-03-15 周六 10:00 - 王老师
2. 2024-03-15 周六 14:00 - 李老师
3. 2024-03-16 周日 09:00 - 王老师

请问您想约哪个时段？是给小明约课吗？
```

### Step 4: 执行预约
用户确认后，调用 `book_class` 完成预约：
- `parent_id`: 家长用户ID
- `student_id`: 学生ID
- `tutor_id`: 老师ID
- `time_slot`: 时间段（格式 YYYY-MM-DD HH:MM）

### Step 5: 反馈结果
预约成功后告知用户：
- 课程时间（用户时区）
- 上课老师
- 学生姓名
- 温馨提示（如上课前检查设备）

## 注意事项

1. **提前12小时**：约课需提前12小时，给老师预留备课时间
2. **单次操作**：每次只处理一节课的预约，批量约课请引导联系顾问
3. **老师偏好**：如用户提到"上次的老师"，先调用 `search_user_preferences` 查询偏好
4. **时间偏好**：如用户说"和之前一样的时间"，先查询用户偏好

## 特殊情况处理

### 无可用时段
```
抱歉，[日期]暂时没有可约的时段。您可以：
1. 尝试其他日期
2. 选择其他老师
3. 联系顾问协助安排
```

### 用户犹豫
不要催促，提供更多信息帮助用户决策：
- 老师简介
- 课程特色
- 上课注意事项
