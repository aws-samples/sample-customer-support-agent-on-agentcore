# 取消课程操作流程

## 触发条件
用户表达取消课程意图，如：
- "我想取消课程"
- "明天的课不上了"
- "帮我取消周六的课"
- "小明那节课请假"

## 标准流程

### Step 1: 查询已约课程
调用 `get_booking_records` 查询用户已预约的课程：
- 使用 `parent_id` 查询
- 如用户指定日期范围，使用 `start_date` 和 `end_date` 筛选

### Step 2: 确认取消课程
展示课程信息并请求用户确认：
- 课程时间
- 老师姓名
- 学生姓名
- 课程类型

**示例回复**：
```
您有以下已约课程：

1. 2024-03-15 周六 10:00 - 王老师 - 小明 - 中文标准版
2. 2024-03-16 周日 14:00 - 李老师 - 小红 - 新加坡数学

请问您要取消哪节课？
```

### Step 3: 执行取消
用户确认后，调用 `cancel_class` 完成取消：
- `parent_id`: 家长用户ID
- `booking_id`: 预约记录ID

### Step 4: 反馈结果
取消成功后告知用户：
- 已取消的课程信息
- 课时已退回账户（如适用）

**示例回复**：
```
好的，已为您取消以下课程：
- 时间：2024-03-15 周六 10:00
- 老师：王老师
- 学生：小明

课时已退回您的账户。
```

## 注意事项

1. **提前2小时**：取消需在上课前2小时操作
2. **单次操作**：每次只处理一节课的取消，批量取消请引导联系顾问
3. **确认必需**：必须等用户确认后才执行取消，不能自动取消
4. **小班课**：小班课取消会消耗请假机会，需提醒用户

## 特殊情况处理

### 时间过期（不足2小时）
```
抱歉，这节课距离上课时间不足2小时，无法在APP内取消。
请您联系顾问协助处理。
```

### 无课程可取消
```
没有查询到您待上的课程。
请问您是想取消哪天的课程？可以告诉我具体日期。
```

### 批量取消请求
```
我只能帮您取消单节课程。如果您想批量取消课程（如"把以后的课都取消"），
请联系顾问协助处理。
```

### 用户反悔
如果用户在确认前改变主意，不要执行取消操作，礼貌回应：
```
好的，那这节课保留不变，有需要随时找我~
```
